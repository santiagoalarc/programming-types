Paradigmas de Programación

Guía introductoria a los estilos fundamentales de programación

Un paradigma de programación es un enfoque o estilo fundamental para diseñar y estructurar programas. Cada paradigma ofrece una forma distinta de pensar sobre la resolución de problemas mediante código. A continuación, se describen los principales paradigmas utilizados en el desarrollo de software.

1. Programación Imperativa

Este es el paradigma más tradicional. Consiste en dar al computador una secuencia de comandos que debe ejecutar paso a paso. Se basa en cambiar el estado del programa mediante asignaciones y estructuras de control como bucles y condicionales.

  • Ejemplo: Lenguajes como C, Pascal y ensamblador.
  • Características: Secuencial, uso de variables, modificaciones explícitas del estado.

int suma = 0; for (int i = 1; i <= 5; i++) { suma += i; }

2. Programación Orientada a Objetos (POO)

La POO organiza el software en objetos, que son instancias de clases. Cada objeto combina datos (atributos) y comportamientos (métodos). Se basa en principios como encapsulamiento, herencia, polimorfismo y abstracción.

  • Ejemplo: Java, C++, Python, Ruby.
  • Beneficios: Reutilización de código, modularidad, mantenimiento más fácil.

class Perro { String raza; void ladrar() { ... } }

3. Programación Funcional

En este paradigma, los programas se construyen mediante la evaluación de funciones matemáticas. Se evita el cambio de estado y los datos inmutables son clave. Las funciones pueden ser tratadas como valores (funciones de primera clase).

  • Ejemplo: Haskell, Lisp, Erlang, también soportado en JavaScript, Python y Scala.
  • Ventajas: Código más predecible, fácil de probar y depurar.

const doble = x => x * 2; const numerosDobles = [1,2,3].map(doble);

4. Programación Declarativa

En lugar de especificar cómo hacer algo (como en imperativo), se describe qué se quiere lograr. Es más abstracto y conciso.

  • Ejemplos: SQL (para consultas), HTML (estructura), Prolog (lógica).
  • Subtipos: Funcional y lógica son formas de programación declarativa.

SELECT nombre FROM usuarios WHERE edad > 18; — Describe qué obtener, no cómo recorrer los datos.

5. Programación Orientada a Eventos

El flujo del programa está determinado por eventos como pulsaciones de teclas, clics del ratón o mensajes entrantes. Muy común en aplicaciones interactivas y sistemas GUI.

  • Uso típico: Aplicaciones web (JavaScript), interfaces gráficas, servidores Node.js.
  • Mecanismo: Se registran "listeners" o manejadores de eventos.

boton.addEventListener('click', () => alert('¡Clic!'));

6. Programación Lógica

Basada en reglas y hechos lógicos. El programador define relaciones y el sistema deduce soluciones automáticamente.

  • Lenguaje principal: Prolog.
  • Aplicaciones: Inteligencia artificial, sistemas expertos, procesamiento del lenguaje natural.

padre(juan, maria). hijo(X,Y) :- padre(Y,X).

Conclusión

No existe un único "mejor" paradigma. La elección depende del problema a resolver, el entorno y las preferencias del equipo. Muchos lenguajes modernos (como Python, JavaScript o Scala) son multi-paradigma, lo que permite combinar ventajas de varios estilos.

Comprender estos paradigmas ayuda a escribir código más claro, eficiente y mantenible, y a elegir las herramientas adecuadas para cada tarea.
